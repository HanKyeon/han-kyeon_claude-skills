# Phase 1 Sanity Check

Phase 1 답변(기본 5 + Deep-dive 옵션 답변)이 모두 모인 뒤, **패턴 추천 직전** 수행하는 모순 감지 단계입니다.

## 원칙

1. **모순만 지적.** 선호 차이는 배제.
2. **해석 2~3개 제시.** "A로 볼 수도 B로 볼 수도 있습니다" 형식.
3. **한 번에 통합 질문.** 재질문 2회 이상 금지. 2회째에도 남는 모순은 사용자 의도로 간주하고 그대로 진행.
4. **통과도 명시.** 모순 없으면 한 줄로 "Sanity check 통과" 보고.

## 모순 감지 룰 7개

### R1. 규모 vs 전문성 축 수
- **조건**: Q5(규모) < Q3(전문성 축 수). 예: 축 5개인데 규모 2 에이전트.
- **재질문**: "독립 평가 축이 <N>개인데 에이전트가 <M>명입니다. 한 에이전트가 여러 축을 겸임합니까, 축 일부를 빼야 합니까?"

### R2. 실패 비용 vs 패턴 후보 불일치
- **조건**: Q4=(c)운영 장애/데이터 손실 인데 Q1 태스크 성격이 Producer-Reviewer 형태가 아님 (예: (e) 동적 분기).
- **재질문**: "실패 비용이 매우 높습니다(Q4=c). (a) Producer-Reviewer로 안전 강화 (b) 현재 패턴에 사람 승인 관문 추가 (c) 이대로 진행(위험 수용) 중 어느 쪽인가요?"

### R3. 태스크 성격 vs 입출력 형태
- **조건**: Q1=(a)선형 변환이라 답했는데 Q2 출력이 단일 리포트(보통 Expert Pool의 산출물).
- **재질문**: "선형 변환이라 하셨는데 출력이 단일 리포트입니다. 실제로 단계가 선형입니까, 여러 관점이 동시에 나와야 하는 것입니까?"

### R4. 전문성 축 겹침
- **조건**: Q3 축 중 2개 이상이 의미상 겹침 (예: "보안"과 "인증/권한", "성능"과 "렌더링 최적화").
- **재질문**: "'<축 A>'와 '<축 B>'가 범위가 겹쳐 보입니다. (a) 병합해 1개 축으로 (b) 범위 명시하여 분리 유지 중 어느 쪽?"

### R5. Pre-scan 발견 기존 에이전트 충돌
- **조건**: `./.claude/agents/`에 이미 있는 에이전트 중 하나와 Q3 축 겹침.
- **재질문**: "기존 에이전트 `<name>`이 `<축>`을 담당합니다. 새 팀에서 이 축은 (a) 제거(기존 재사용) (b) 분리(이름 다르게) (c) 기존 에이전트 교체 중 어느 쪽?"

### R6. Deep-dive 답변 모순 (옵션 답변이 있을 때만)
- **조건**: 권한이 `Read only`인데 실패 처리가 `자동 롤백`. 읽기만으로는 롤백 불가.
- **재질문**: "권한을 읽기 전용으로 두셨는데 롤백이 필요합니다. 롤백은 오케스트레이터가 담당합니까, 에이전트에 쓰기 권한을 부여합니까?"

### R7. 에이전트 간 통신 vs 실험 플래그 인식
- **조건**: Deep-dive에서 "에이전트 간 직접 메시지 교환 필요"라고 답했는데 사용자가 `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` 필요성을 모름.
- **재질문 (정보 알림 + 확인)**: "에이전트 간 직접 통신이 필요한 설계입니다. `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` 환경변수가 필요합니다. (a) 알겠음, 그대로 진행 (b) 오케스트레이터 경유만 하는 설계로 변경 중 어느 쪽?"

## 실행 순서

```
1. Phase 1 기본 답변 + Deep-dive 답변(있으면) 확보
2. R1~R7 각각 체크. Deep-dive 미수행 시 R6, R7 skip.
3. 모순 건수:
   - 0건 → "Sanity check 통과" + 요약 카드 + 패턴 추천
   - 1~2건 → 해당 지점만 재질문 (한 번에 묶어서)
   - 3건+ → 드문 케이스. 답변 근본 재검토 제안.
4. 2회째 검사 후 남은 모순은 사용자 의도로 간주.
```

## 출력 템플릿

### 통과 시
```
✅ Sanity check 통과 — Phase 1 답변 상충 없음. 요약 카드와 패턴 추천으로 진행합니다.
```

### 모순 발견 시
```
⚠️ Sanity check — 아래 지점 재확인 부탁드립니다.

1. [R2] Q4(실패 비용)가 '운영 장애'인데 현재 추천 패턴은 Producer-Reviewer가 아닙니다.
   → (a) Producer-Reviewer로 변경 (b) 사람 승인 관문 추가 (c) 그대로 진행

2. [R4] 축 '<A>'와 '<B>'가 범위가 겹칩니다.
   → (a) 병합 (b) 범위 명시하여 분리
```

## 하지 말 것

- 모순이 아닌 **설계 제안**을 이 단계에서 내지 말 것 (그건 Phase 3 Agent Roster에서).
- 자동 수정 금지 — 반드시 질문으로.
- 7개 룰을 모두 설명하지 말 것 — 걸린 항목만 노출.
