# Legacy Tolerance Rules

리팩터링 중 발견한 "이것도 이상한데" 문제를 **이번 PR에서 고치지 않는** 원칙과 대응법입니다.

## 왜 허용하는가

- **Scope 폭발 방지**: 한 번 손대기 시작하면 끝이 없다. 리팩터링이 영원히 안 끝난다.
- **리뷰 부담**: PR이 커질수록 결함 발견율이 떨어진다.
- **롤백 난이도**: 여러 문제를 한 번에 고치면 어디서 문제가 생겼는지 특정하기 어렵다.
- **심리적 안전**: 작업자가 "완벽하게 고쳐야 한다"는 압박에서 해방되어야 PR이 머지된다.

## 발견 시 처리 프로토콜

발견한 legacy 문제는 **고치지 말고 기록**한다.

### 단계 1: 즉시 판정

| 유형 | 대응 |
|---|---|
| **Critical** (보안 취약점, 데이터 손실 가능성, 프로덕션 다운 위험) | 현재 작업 중단. 사용자에게 즉시 보고 → 별도 hotfix PR |
| **High** (주요 버그, 성능 저하, 접근성 심각 위반) | 이번 PR 범위 밖이면 Issue 생성, 이번 PR은 계속 |
| **Medium/Low** (코드 스타일, 네이밍, 경미한 비효율) | TODO 주석 없이 Issue 또는 follow-up 리스트로 기록 |

### 단계 2: 기록 형식

PR 설명에 "Found but not fixed" 섹션을 둔다:

```markdown
## Found but not fixed (follow-up)

- `src/foo/bar.ts:123` — `any` 타입 사용. [#456](issue-link)
- `src/baz.tsx:45` — useEffect 의존성 누락. [#457](issue-link)
- `src/qux.ts` — 전체 파일이 1000줄 넘음. [#458](issue-link)
```

### 단계 3: 이슈 트래커 활용

TODO 주석은 **지양**한다. 이유:
- 커밋에 묻혀 사라진다
- 우선순위가 안 보인다
- 담당자가 없다

대신 Issue 트래커 (GitHub Issues, Jira, Linear 등)에 생성하고 라벨 `tech-debt`, `legacy`, `follow-up` 부여.

## 예외 — 고쳐도 되는 경우

Scope 밖이어도 **고쳐도 되는** 예외:

1. **현재 변경이 그 legacy 때문에 막힘** — 필수적인 경우만. 최소 수정.
2. **한 줄짜리 명백한 타입 오류** — 5분 내 수정 + 커밋 분리.
3. **현재 파일 내부의 죽은 코드** — 수정 중인 파일에 한해 제거.

위 예외도 **별도 커밋**으로 분리한다. `chore:` 또는 `fix:` prefix로.

## AI 에이전트에게 강제할 규칙

리팩터링 스킬이 활성화되면 에이전트는:

1. **Scope 밖 수정 전 반드시 승인 요청**. "이것도 고쳐도 될까요?"를 물음.
2. **발견 사항을 자동으로 누적**. PR 설명 초안에 "Found but not fixed" 섹션 생성.
3. **"이왕 보는 김에" 충동 억제**. 한 PR의 변경은 하나의 주제로 유지.

## 자주 하는 실수

| 실수 | 교정 |
|---|---|
| "이 함수도 엉망이네, 같이 고침" | 별도 PR로. 현재 PR에 섞지 말 것. |
| `// TODO: fix later` 남김 | Issue 생성. TODO 없이. |
| "전체 파일 통째로 prettier" | 별도 chore 커밋. |
| "쓰지 않는 import 정리도 같이" | 별도 chore 커밋. |
| "무심코 변수명 개선" | Scope 밖이면 되돌리기. |

## 사용자와의 대화 예시

**AI**:
> 리팩터링 범위를 `src/components/patient-table/Bp.tsx`의 컴포넌트 구조 개선으로 확정했습니다.
>
> 작업 중 **Scope 밖** 이슈 2개를 발견했습니다:
> 1. `Bt.tsx:34` — 같은 컴포넌트에서 동일한 패턴이 반복되고 있습니다. 이 PR 다음 단계 후보.
> 2. `Pr.tsx:12` — `any` 타입 사용.
>
> 둘 다 이번 PR에서는 **건드리지 않겠습니다.** follow-up 리스트에 기록했습니다. 진행해도 될까요?
